## 剩余风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 极窄屏（<320px）步骤标签可能被截断 | 低 | `.steps` 有 overflow-x auto，可横向滚动；已设 420px 断点优化 |
| 复制清单在非 HTTPS 环境可能失败 | 低 | 已有 try/catch 兜底，失败时显示红色"复制失败，请手动选择文本复制" |
| 打印样式在不同浏览器可能有细微差异 | 低 | 使用 `!important` 强制隐藏，主流浏览器兼容 |
| 未配置 Key 时 AI 状态为 skipped | 预期行为 | 非风险，设计如此；本地规则预审仍可演示 |
| 真实 AI 联网未验证 | 低 | `live-ai-smoke` 默认跳过；用户可手动 `LIVE_AI_SMOKE=1` 测试 |

## 最终演示建议

1. **演示前准备**：运行 `npm run dev`，确认前端（5173）和后端（3001）均启动；打开 http://localhost:5173。
2. **演示流程**：点击首页"一键演示场景"→ 跳到第 5 步 → 点击"生成预审结果" → 展示办事准备度/缺失材料/老人友好清单 → 点击"复制清单" → 点击"打印页面"。
3. **讲解重点**：强调"结果不代表官方审核通过"、未配置 Key 时本地规则仍可演示、配置 Key 后可启用真实 AI 增强。
4. **移动端演示**：可使用浏览器 DevTools 切换到 390px 宽度，演示移动端布局适配。
5. **安全提示**：演示过程中不展示 .env 内容、不展示 API Key、不执行真实联网调用。
